# BrandWiki Schema — 行为规则与页面模板

本文件是 BrandWiki 的第三层架构（行为规则层），定义目录结构、页面模板、操作规则和质量标准。AI 作为 Wiki 维护者必须严格遵守本文件中的所有规则。

## 目录结构

```
{WIKI_ROOT}/                   # 路径从 ~/.qoderworkcn/brand-wiki-path 读取
├── SCHEMA.md                  # 本文件：行为规则与模板
├── _log.md                    # 全局操作日志（只追加，不修改历史）
│
├── brands/                    # 品牌层：每个品牌一个独立目录
│   └── {品牌名}/
│       ├── _meta.md           # 品牌基本信息 + 报告/对话时间线
│       ├── _dashboard.md      # 核心指标趋势表（每次追加数据行）
│       ├── people.md          # 人维度：用户经营诊断
│       ├── products.md        # 品维度：商品/菜品/产品分析
│       ├── store.md           # 店维度：门店/渠道/区域分析
│       └── solutions.md       # 数字化解决方案（必须关联痛点）
│
├── domain/                    # 领域层：跨品牌通用知识（需人工确认才入库）
│   ├── _index.md              # 领域知识索引
│   ├── frameworks/            # 分析框架与方法论
│   ├── patterns/              # 经验证的有效打法
│   ├── benchmarks/            # 行业基准与对标数据
│   └── concepts/              # 业务概念与术语定义
│
└── raw/                       # 原始文件存档
    ├── uploads/               # 用户上传的文档原件
    └── reports/               # skill 生成的 HTML 报告存档
```

## 操作规则

### Ingest（入库）判断标准

以下三个条件满足任一即视为"值得入库"：
1. **包含新的业务事实或判断**：非纯操作类对话，产生了有价值的分析结论
2. **对未来对话有参考价值**：下次涉及同品牌/同话题时，AI 知道这些会更好
3. **与已有知识矛盾或补充**：纠正了 Wiki 旧结论或提供了新数据点

以下不入库：
- 纯操作指令（改颜色、调格式、导出文件）
- 一次性信息查询（天气、日期、简单计算）
- 无结论的闲聊

### Ingest 流程

1. 对话尾声或报告完成后，AI 静默评估是否满足入库标准
2. 如满足，生成简短摘要卡片展示给用户
3. 用户确认（全部/部分/跳过）
4. AI 执行文件写入：
   - 更新对应品牌的维度页面（追加"历史演变"行、更新"当前结论"）
   - 更新 _dashboard.md（如有新指标数据）
   - 追加 _log.md（操作记录）
   - 如有上传文档，存原始文件到 raw/uploads/
5. 如涉及跨品牌通用知识，单独提议是否存入 domain/

### Recall（调取）流程

**报告生成场景（brand-review-report skill 之前）：**
1. 识别当前品牌名
2. 读取 brands/{品牌名}/ 下所有页面
3. 将历史诊断、指标趋势、已有方案作为上下文注入报告生成

**业务问答场景：**
1. 识别问题涉及的品牌和维度
2. 读取对应品牌页面 + domain 中相关知识
3. 结合 Wiki 知识 + 实时分析给出回答

### _log.md 记录格式

```
[YYYY-MM-DD] INGEST | {品牌名}/{维度} | {一句话摘要} | 来源：{报告/对话/上传}
[YYYY-MM-DD] RECALL | {品牌名} | 用于：{报告生成/业务问答}
[YYYY-MM-DD] LINT   | 发现 {N} 个问题 | {问题概要}
```

### 质量规则

1. **方案必须关联痛点**：solutions.md 中每条方案必须关联至少一个维度页面的具体痛点
2. **每个维度必须有当前结论**：维度页面的"当前结论"段落不能为空
3. **历史演变只追加不删除**：保留完整认知演进轨迹，即使旧结论被推翻也标记而非删除
4. **跨品牌知识标注来源**：domain/ 中的知识必须标注支撑它的品牌实践或文档来源
